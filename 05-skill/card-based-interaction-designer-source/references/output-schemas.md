# Output Schemas

- Deck name:
- Purpose:
- Audience:

- Relationship dynamics:
- Deck type:
- Disclosure level:
- Facilitation mode:
- Capture mode:
- Topics to include:
- Topics to avoid:

- Output format:
- Safety notes:

## Card table

Use these fields for automation, printing, or review:

| Card ID | Deck | Deck Type | Card Type | Category | Tone Level | Disclosure Level | Risk Level | Icon | Front Label | Prompt | Options | Follow-up | Facilitator Note | Pass-Friendly Wording | Review Status | Revision Note |

## Review table

Use these fields for reviewing an existing deck:

| Card ID | Card Type | Category | Prompt Summary | Clarity | Engagement | Usefulness | Safety | Audience Fit | Actionability | Flag | Issue | Suggested Revision |

Scoring can be numeric or labelled:

- Strong.
- Good.
- Needs work.
- Risky.
- Remove.

## Instruction sheet

Use this structure:

1. What this deck is for.
2. How to play or use it.
3. Passing rule.
4. Kindness / no-weaponising rule.
5. Timing options.
6. Group-size options.
7. Optional debrief.
8. What not to use it for.

## Facilitator guide

Use this structure:

1. Purpose.
2. Room setup.
3. Group agreements.
4. How to introduce the activity.
5. Timing plan.
6. What to watch for.
7. How to handle passes or discomfort.
8. Debrief questions.
9. Capture rules.
10. Follow-up actions.

## Revision log

Use this structure:

| Version | Date | Audience | Change Made | Reason | Reviewed By | Known Issues | Next Test |
